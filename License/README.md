# 라이선싱 시스템 사용 가이드

## 개요
이 라이선싱 시스템은 하드웨어 ID 기반의 암호화된 라이선스 파일을 사용하여 프로그램의 불법 복제를 방지합니다.

## 작동 방식

### 1. 하드웨어 ID 생성
- 프로그램이 실행되는 컴퓨터의 고유 정보를 수집합니다:
  - MAC 주소 (네트워크 어댑터)
  - CPU ID (프로세서 고유 ID)
  - 하드디스크 시리얼 번호
  - 컴퓨터 이름
- 이 정보들을 조합하여 SHA256 해시로 고유한 하드웨어 ID를 생성합니다.

### 2. 라이선스 파일 생성 (관리자)
1. **RSA 키 쌍 생성** (최초 1회만)
   - `KeyGenerator.GenerateAndSaveKeys()` 메서드를 호출하여 키를 생성합니다.
   - 생성된 키는 `Keys/public_key.xml`과 `Keys/private_key.xml`에 저장됩니다.
   - **중요**: `private_key.xml`은 절대 배포하지 마세요. 관리자만 보관합니다.

2. **라이선스 생성**
   - 고객의 컴퓨터에서 하드웨어 ID를 확인합니다.
   - `WindowLicenseGenerator` 윈도우를 사용하거나 `LicenseGenerator.GenerateLicenseFile()` 메서드를 호출합니다.
   - 생성된 라이선스 파일(`license.lic`)을 고객에게 제공합니다.
   - **보안**: 라이선스 생성 후 `private_key.xml`이 자동으로 삭제됩니다 (기본 설정).
     - 개인키가 유출되면 누구나 라이선스를 생성할 수 있으므로 반드시 삭제하거나 안전한 곳에 백업하세요.

### 3. 라이선스 검증 (프로그램 실행 시)
- 프로그램 시작 시 `App.xaml.cs`의 `OnStartup` 메서드에서 자동으로 검증합니다.
- 검증 항목:
  1. 라이선스 파일 존재 여부
  2. RSA 서명 검증 (위조 방지)
  3. 라이선스 파일 복호화 가능 여부
  4. 하드웨어 ID 일치 여부
  5. **시간 조작 검증**:
     - 발급일 검증 (발급일보다 이전 시간이면 차단)
     - 시간 역행 감지 (1시간 이상 역행하면 차단)
     - 비정상적인 시간 앞당김 감지 (1년 이상 앞서면 경고)
  6. 만료일 검증 (발급일 기준으로 계산)
  7. 마지막 실행 시간 저장 (암호화)

### 4. 암호화 방식
- **AES 암호화**: 라이선스 데이터는 AES-256으로 암호화됩니다.
- **RSA 서명**: 암호화된 데이터에 RSA 서명을 추가하여 위조를 방지합니다.
- **하드웨어 ID 기반 키**: AES 키는 하드웨어 ID에서 파생되므로, 하드웨어가 변경되면 복호화할 수 없습니다.

## 파일 구조

```
프로젝트/
├── License/
│   ├── HardwareIdGenerator.cs      # 하드웨어 ID 생성
│   ├── LicenseData.cs               # 라이선스 데이터 구조
│   ├── LicenseManager.cs            # 라이선스 검증
│   ├── LicenseGenerator.cs          # 라이선스 생성
│   ├── KeyGenerator.cs              # RSA 키 생성
│   ├── WindowLicenseGenerator.xaml  # 라이선스 생성 UI
│   └── WindowLicenseGenerator.xaml.cs
└── Keys/                            # (생성됨) RSA 키 저장 폴더
    ├── public_key.xml               # 공개키 (프로그램에 포함)
    └── private_key.xml              # 개인키 (관리자만 보관)
```

## 라이선스 파일 저장 위치

### 프로그램 실행 시:
- **라이선스 파일**: `[프로그램 실행 파일 경로]/license.lic`
- **공개키 파일**: `[프로그램 실행 파일 경로]/Keys/public_key.xml`

### 예시:
```
C:\Program Files\YourApp\
├── YourApp.exe
├── license.lic          ← 여기에 저장
└── Keys\
    └── public_key.xml   ← 여기에 저장
```

## 사용 방법

### 1. 초기 설정 (최초 1회)

#### 방법 1: 코드에서 키 생성
```csharp
// 임시로 어딘가에서 호출 (예: 버튼 클릭 이벤트)
KeyGenerator.GenerateAndSaveKeys();
```

**⚠️ 중요**: 
- 기존 키가 있으면 경고 메시지가 표시됩니다.
- 새로운 키를 생성하면 **기존에 생성된 모든 라이선스가 무효화**됩니다.
- 키를 다시 생성하기 전에 반드시 기존 키를 백업하세요.

#### 방법 2: 별도 유틸리티 프로그램 생성
라이선스 생성 전용 프로그램을 만들어서 키를 생성합니다.

### 2. 라이선스 생성

#### 방법 1: UI 사용
```csharp
WindowLicenseGenerator window = new WindowLicenseGenerator();
window.ShowDialog();
```

#### 방법 2: 코드에서 직접 생성
```csharp
string hardwareId = "고객의_하드웨어_ID";
DateTime? expiryDate = new DateTime(2025, 12, 31); // 또는 null (무제한)
string customerName = "고객명";

LicenseGenerator.GenerateLicenseFile(hardwareId, expiryDate, customerName);
```

### 3. 라이선스 배포
생성된 `license.lic` 파일을 고객의 프로그램 설치 폴더에 복사합니다.

## 보안 고려사항

1. **개인키 보안**
   - `private_key.xml`은 절대 배포하지 마세요.
   - 관리자만 보관하고 안전한 곳에 백업하세요.
   - 개인키가 유출되면 누구나 라이선스를 생성할 수 있습니다.
   - **라이선스 생성 후 자동 삭제**: 보안을 위해 라이선스 생성 후 `private_key.xml`이 자동으로 삭제됩니다.
     - 추가 라이선스가 필요하면 백업된 개인키를 복원하거나, 새로운 키 쌍을 생성해야 합니다.

2. **공개키 배포**
   - `public_key.xml`은 프로그램과 함께 배포됩니다.
   - 프로그램에 포함되어 있어도 안전합니다 (공개키이므로).
   - 공개키는 서명 검증에만 사용되므로 유출되어도 문제없습니다.

3. **라이선스 파일**
   - AES로 암호화되어 있어 내용을 직접 읽을 수 없습니다.
   - RSA 서명으로 위조를 방지합니다.
   - 하드웨어 ID 기반 키를 사용하므로 파일 복사로 다른 컴퓨터에 사용할 수 없습니다.
   - 하드웨어가 변경되면 자동으로 복호화가 실패합니다.

4. **시간 조작 방지**
   - 마지막 실행 시간을 암호화하여 저장합니다 (`Keys/.lastrun`).
   - 시간 역행 감지: 시스템 시간을 과거로 되돌리면 차단됩니다.
   - 발급일 검증: 라이선스 발급일보다 이전 시간이면 차단됩니다.
   - 만료일은 발급일 기준으로 계산되므로, 시간을 되돌려도 만료일이 연장되지 않습니다.

## 문제 해결

### 라이선스 검증 실패 시
1. 라이선스 파일이 올바른 위치에 있는지 확인
2. 하드웨어 ID가 일치하는지 확인
3. 만료일이 지나지 않았는지 확인
4. 공개키 파일이 존재하는지 확인

### 하드웨어 변경 시
고객이 컴퓨터를 변경하거나 주요 하드웨어를 교체한 경우:
1. 새로운 하드웨어 ID를 확인
2. 새로운 라이선스 파일을 생성하여 제공

## 추가 기능 확장 가능

- 온라인 라이선스 서버 연동
- 라이선스 갱신 기능
- 기능별 라이선스 제한
- 사용 횟수 제한
- 라이선스 활성화/비활성화

## 주의사항

- 이 시스템은 기본적인 보호를 제공하지만, 완벽한 보안을 보장하지는 않습니다.
- 중요한 상용 프로그램의 경우 전문 라이선싱 솔루션 사용을 고려하세요.
- RSA 키는 2048비트를 사용하며, 더 강력한 보안이 필요하면 4096비트로 변경 가능합니다.
