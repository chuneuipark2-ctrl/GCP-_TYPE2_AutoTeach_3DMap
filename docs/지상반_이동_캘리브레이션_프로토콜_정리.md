# 지상반 크레인 이동 · 캘리브레이션 프로토콜 정리

> 작성: 2026-07-07 · 대상 코드: `gcp_type2-TP2_VER1` (지상반 WPF)
> 근거: Vexi 통신 프로토콜 Rev.92 (2026-04-24, 양일규) + 실장비 로그 + 소스 대조
> 용도: 팀 공유 — "오토티칭 때 크레인이 어떤 프로토콜로 움직이는지 + 캘리브레이션 로직 전체"

모든 크레인 통신은 **UDP** (지상반 ↔ SRM MCU), 비전 관련은 **HTTP** (지상반 ↔ 비전서버 :3080, 라이다 서비스 :9000).

## 1. 오토티칭 중 셀 이동 — 반자동(0x41) 경로

티칭 루프(Phase 2)에서 셀→셀 이동에 쓰는 경로. **mm를 직접 보내지 않는다.**

### 명령 체인
| 순서 | 프로토콜 | 내용 |
|---|---|---|
| 런 시작 1회 | `0x0058` 모드설정 | 크레인 자동모드 전환 (cmd=2) |
| 런 시작 1회 | `0x0050` 시작 | 시작 ON (data=1). 런 종료/중단 시 OFF(0) = 무장해제 |
| 셀마다 | `0x0041` 반송명령 | **이동 명령(작업코드 0x01)** — 아래 패킷 구조 |
| 상시 | `0x0030` 상태조회 | 도착 판정용 폴링 (위치·정위치·작업상태) |

### 0x0041 패킷 내용 (이동 명령일 때)
```
작업코드(1) = 0x01(이동) | 옵션(1) | 작업번호(4)
From: Station/Row/Bay(2)/Lev = 전부 0 (이동 명령은 To만 사용)
To:   Station(1) / Row(1) / Bay(2, LE) / Lev(1)
GoodsType(1) + Fork2 영역(동일 구조) + reserved
```
- **핵심: 셀 좌표(Row/Bay/Lev)만 전송. mm 필드 없음.**
- 실제 목표 mm는 **크레인 내부 셀 테이블**이 결정한다. 이 테이블은 `0x0094`(셀설정 조회)로 읽고 `0x0095`(셀위치 설정)로 쓴다 — 티칭 결과 "반영"이 바로 0x95 쓰기.
- 송신 조건(지상반 앱): 시작 상태(dSt1StartSt>0) + 지상반 송신모드 반자동(gcpTxMode==2).

### 도착 판정 (앱 측, 0x30 응답 기반)
- 위치: `trav.curPos`(패킷 offset 95, Int32 mm) / `lift.curPos`(offset 111)
- 판정: (주행·승강 정위치 비트 ON **또는** 작업완료 상태) **그리고** 목표 ±100mm 이내
- 이후 정밀 확인 단계에서 ±1mm 재검(차이 크면 경고만 하고 진행)
- 안전: 15초 무변화 stall 조기중단 + 위치 envelope 이탈 시 즉시 시작 OFF(0x50) + 런 중단

### ★ 승강이 항상 목표보다 30mm 낮게 서는 이유 (Δ30)
셀값 3030 지시 → 크레인 3000 도착, 셀값 1000 → 970 도착이 **매번** 재현되는데, 이는 크레인 파라미터
**`0x00A5` 응답의 "승강 하강 정위치 높이 설정"(Signed 1B, 범위 -100~0, 전체 셀 공통 적용)이 -30mm로 설정**돼 있기 때문.
실측 근거: 0xA5 파라미터 덤프 offset 278 = `0xE2` = **-30**. (인접 값도 스펙과 일치: 정위치 오차범위=3, 인버터 사용=1, 상승 정위치=50)
→ 앱 버그가 아니라 크레인 설정값. 바꾸려면 0xA6 쓰기(해당 필드) 또는 Vexi 설정 화면.

## 2. 캘리브레이션 — 보수위치(0x59) mm 정밀 이동 경로

캘리브 스윕은 ±5/10/15mm 같은 **임의 mm 이동**이 필요해서 0x41(셀 단위)로는 불가능 → 보수위치 경로 사용.

### 크레인 이동 명령 체인 (스윕 포인트마다 반복)
| 순서 | 프로토콜 | 내용 |
|---|---|---|
| ① | `0x0058` 모드설정 | 셋업모드(cmd=1). **2026-07-07부터: 현재 모드에서 먼저 시도하고, 거부(NACK 10)될 때만 셋업 폴백** |
| ② | `0x00A3` / `0x00A5` | 주행/승강 드라이브 설정 **조회** (파라미터 블록 통째로 읽기 — 모드 무관) |
| ③ | `0x00A4` / `0x00A6` | 주행/승강 드라이브 설정 **변경**: 읽은 블록에서 속도그룹 전체를 20m/min으로 통일 + **보수 위치 필드에 목표 mm** 기입 후 재전송 (제어플래그로 적용 그룹 선택) |
| ④ | `0x0059` 보수위치 이동 | 본문 없음(빈 패킷) — ③에서 써둔 보수위치로 이동 개시. 응답(0x8059)은 거의 안 옴 |
| ⑤ | `0x0030` 폴링 | curPos로 도착 판정 (±1mm + 120ms 안정화, 15s stall 조기중단, 최대 120s) |

### 캘리브레이션 전체 시퀀스 (CALIB 버튼)
```
1. 사전 경고: 현재 승강이 1015mm 미만이면 "스윕 거부 예상" 다이얼로그 (강행 가능)
2. HTTP GET  /api/gc/cmd/calibration/status?camera_id=…   ← 기존 캘리브 확인
3. HTTP POST (세팅 이미지 정리 — setting_images 삭제)
4. HTTP POST /api/gc/cmd/lidar/scan_start                  ← 라이다 스캔(빈 JSON = 20프레임)
5. 주행(X) 스윕: 기준 trav ±15mm, 5mm 간격 7포인트 (승강 고정)
   포인트마다: [위 크레인 이동 체인 ②~⑤] → 위치확인 → HTTP POST /api/gc/cmd/{cam}/calibration/capture
6. 승강(Z) 스윕: 기준 lift ±15mm 7포인트 (주행 고정) — 동일 반복
7. HTTP POST /api/gc/cmd/{cam}/calibration/inference (compute)
   ★ 서버 요구조건: X 7장 + Z 7장 (각 축: 아래 3 + 정위치 1 + 위 3). 한 장이라도 모자라면 실패
8. 결과: offset_x(주행 보정 mm/px), offset_y(현재 서버 구현상 항상 0) 파일 저장
```

### 알려진 MCU 펌웨어 제약 (구펌웨어 4.4 기준 — 실장비 확인)
1. **셋업모드 요구**: 0xA4/0xA6 쓰기는 셋업모드 아니면 NACK(제어실패코드 **10 = 셋업모드 아님**, 프로토콜 문서 명시).
2. **승강 보수위치 범위 검증 버그**: 0x59에서 **승강** 보수위치를 **주행** ManualOp 범위(수동운전 시작~끝, 당사 1000~25300mm)로 검증 → 승강 목표 <1000mm면 `0x8059 result=8`로 거부(이동 없음).
   - 실측 근거: 승강 자체 ManualOp=950~5150, 승강 SoftLimit=980~5150(Det=0 비활성)이라 985mm 거부를 승강 쪽 어떤 한계로도 설명 불가. 거부 문턱이 정확히 주행 ManualOp Start(1000)와 일치.
   - 공식 문서에는 0x8059 result가 0/1만 정의됨 — 2~9 코드 의미는 벤더 확인 필요.
   - 영향: 승강 1000mm(레벨1 부근)에서 캘리브 시 Z 아래쪽 3포인트 거부 → 4/7장 → compute 실패. **회피: 승강 1015mm 이상에서 실행.**
3. 지상반 앱은 2026-07-07부터 두 제약 모두 "선차단"하지 않고 **실제로 명령을 보내서 신펌웨어 수정 여부를 응답으로 확인**한다(거부되어도 이동은 없으므로 안전). 로그 판독:
   - `0xA4 … NACK result=10` → 모드 제약 잔존(구펌웨어) → 자동으로 셋업 폴백
   - `0x59 result=8` → 승강범위 버그 잔존 → 해당 포인트 스킵(셋업 재시도 무의미)

## 3. 두 경로 비교 요약

| | 티칭 이동 | 캘리브 스윕 이동 |
|---|---|---|
| 명령 | `0x41` 반송명령(이동) | `0xA4/0xA6` 쓰기 + `0x59` |
| 위치 지정 | 셀 좌표(Row/Bay/Lev) — mm 불가 | **임의 mm** (보수위치 필드) |
| 목표 mm 결정 | 크레인 내부 셀테이블(0x94/0x95) | 지상반이 직접 지정 |
| 요구 모드 | 자동모드 + 시작 ON + 지상반 반자동 송신 | 셋업모드(구펌웨어) / 신펌웨어 완화 확인 중 |
| 속도 | 크레인 자동운전 속도 파라미터 | 0xA4/A6로 전 그룹 20m/min 강제(저속 정밀) |
| 승강 하한 | 셀테이블 값 - 30mm(하강 정위치 offset)까지 내려감 | 구펌웨어 버그로 <1000mm 거부 |
| 도착 판정 | 정위치 비트 + ±100mm → ±1mm 재검 | curPos ±1mm + 안정화 |
