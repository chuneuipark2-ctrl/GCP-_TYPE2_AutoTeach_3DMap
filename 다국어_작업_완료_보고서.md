## 다음 단계

### 1. XAML 파일 업데이트 (우선순위: 높음)
- 모든 하드코딩된 한국어 텍스트를 `{lx:Loc 키이름}` 형식으로 변경
- 예: `Content="설정"` → `Content="{lx:Loc 설정}"`

### 2. 코드 내 한국어 텍스트 업데이트 (우선순위: 높음)
- MessageBox.Show 등의 하드코딩된 한국어를 리소스 키로 변경
- 예: `MessageBox.Show("오류", "오류")` → `MessageBox.Show(Properties.Resources.오류, Properties.Resources.오류)`

### 3. 언어 전환 기능 구현 (우선순위: 중간)
- TranslationSource.CurrentCulture 설정을 통한 런타임 언어 전환
- 언어 설정 UI와 연동

### 4. 테스트 및 검증 (우선순위: 중간)
- 각 언어별 UI 표시 테스트
- 번역 품질 검증 및 수정

### 5. 추가 언어 지원 (우선순위: 낮음)
- 일본어, 베트남어 등 추가 언어 리소스 파일 생성

- **기존 인프라 활용**: TranslationSource, LocExtension 클래스 재사용

- **확장성**: 새로운 언어 추가 시 쉽게 확장 가능
- **유지보수성**: 중앙화된 리소스 관리로 유지보수 용이
